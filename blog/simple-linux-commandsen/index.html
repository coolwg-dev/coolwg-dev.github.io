Simple Linux Commands: A Practical Starter Guide

📅 2026-04-01

Why Learn Simple Linux Commands

Many development and data workflows eventually touch a Linux system: cloud servers, containers, WSL on Windows, or a remote machine you log into over ssh. Being comfortable with a small set of reliable commands makes you faster, safer, and less dependent on graphical tools.

This guide focuses on a pragmatic subset of commands you can memorise and actually use. The goal is not to know every option, but to build a mental model of how the shell sees files, processes, and text.

Getting Comfortable with the Shell

When you open a terminal, you typically land in your home directory. Start by learning how to see where you are and what is around you.

  • pwd – print working directory. Shows the absolute path of your current location.
  • ls – list files in the current directory. Use ls -l for details, ls -a to include hidden files.
  • cd path/ – change directory. Use cd .. to go up one level and cd ~ to return to your home.
  • clear – clear the terminal screen without changing your location.

A useful habit is to run pwd and ls before you execute commands that change or remove files, just to confirm you are in the directory you think you are.

Working with Files and Directories

Most day-to-day work is about creating, copying, moving, and inspecting files and folders.

  • mkdir my-folder – create a new directory.
  • rmdir empty-folder – remove an empty directory.
  • cp source.txt dest.txt – copy a file. Use cp -r src/ dest/ for directories.
  • mv old-name.txt new-name.txt – move or rename a file.
  • rm file.txt – remove a file.

Be careful with recursive deletion: rm -rf will remove directories and their contents without asking. As a beginner, avoid running rm -rf on paths you are not absolutely sure about. Prefer to inspect with ls and remove smaller pieces first.

To quickly look at text files:

  • cat file.txt – print the whole file to the terminal.
  • less file.txt – open a scrollable viewer. Use the arrow keys to navigate and q to quit.
  • head -n 20 file.txt – show the first 20 lines.
  • tail -n 20 file.txt – show the last 20 lines.

Inspecting the System and Processes

You often need to answer questions such as “what is running?” or “what is using disk space?”. These commands give you a quick view.

  • whoami – show the current user.
  • uname -a – print basic system information (kernel and architecture).
  • df -h – display disk usage by filesystem in human-readable units.
  • free -h – show memory (RAM) usage.
  • ps aux – list running processes in detail.
  • top or htop – interactive view of processes and resource usage.
  • kill PID – send a signal to a process by its numeric ID (you can find this from ps or top).

When learning, practice observing processes rather than killing them aggressively. On shared or production systems, coordinate with others before stopping long-running services.

Finding Things and Reading Help

Once you know how to move around, the next step is finding files and understanding what commands do.

  • find . -name "pattern" – search for files under the current directory. For example, find . -name "*.log".
  • grep "text" file.txt – search for lines containing a pattern. Add -i for case-insensitive matches.
  • command --help – show a short help summary for many commands.
  • man command – open the manual page with full documentation. Use the arrow keys or / to search, then q to quit.

Getting into the habit of reading --help output and manual pages will gradually make you independent of copy-paste snippets from the web.

Putting It Together Safely

You do not need to memorise every option on day one. Focus on a core toolkit: pwd, ls, cd, mkdir, cp, mv, rm, cat, less, tail, grep, find, df, free, ps, top, and man.

The safest place to experiment is a disposable environment such as a virtual machine, a container, or a non-critical WSL distribution on your own machine. Once you are comfortable, you can apply the same commands on servers with more confidence and discipline.
